# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported Until |
| ------- | --------------- |
| 0.12.x  | 2027-07-01      |
| 0.11.x  | 2027-01-01      |
| 0.10.x  | 2027-01-01      |
| < 0.10  | Unsupported     |

## Reporting a Vulnerability

If you discover a security vulnerability within AIOS Companion, please
report it immediately via one of the following channels:

- **GitHub Security Advisory**: Use the "Report a vulnerability" button
  in the repository's Security tab (preferred).
- Prefer private disclosure; do not open a public issue for sensitive reports.

Please include:
- A clear description of the issue.
- Steps to reproduce.
- Any relevant logs, screenshots, or proof-of-concept code.
- Your contact information (optional but helpful for follow‑up).

We will acknowledge receipt of your report within **48 hours** and
provide an initial assessment within **7 business days**.

## What Happens Next?

1. **Triage** – We verify the issue and determine its severity.
2. **Investigation** – We work with you (if needed) to understand the
   vulnerability and its impact.
3. **Fix Development** – We prepare a fix in a private branch.
4. **Coordination** – We discuss disclosure timelines and whether a CVE
   should be requested.
5. **Release** – We publish the fix as a security release, updating the
   changelog and notifying users via our usual channels.
6. **Credit** – We will publicly acknowledge your contribution (unless you
   prefer to remain anonymous).

## Preferred Languages

We prefer communications in English, but we will make reasonable efforts
to accommodate other languages.

Thank you for helping keep AIOS Companion safe for everyone.
